What is XML?

A markup language called Extensible Markup Language (XML) provides rules for describing any type of data. As opposed to other programming languages, XML cannot do computations by itself. Instead, organized data may be managed using any computer language or application. It is ideal for data sharing across different systems since it is intelligible to both humans and machines. Web development, configuration files, data storage, and web services like SOAP all commonly use XML. It supports custom tags, follows a hierarchical structure like a tree, and may be verified using XML Schema or DTD to guarantee data accuracy and consistency. XML also promotes data portability by allowing systems with different platforms and technologies to exchange information reliably. Its extensibility makes it suitable for a wide range of industries, from finance and healthcare to publishing and telecommunications.

XML Building Blocks

The building block of XML lies within its formulation, what is it made of and what the starting point is of the language.

XML Declaration

XML declaration contains details that prepare an XML processor to parse the XML document. It is optional, but when used, it must appear in the first line of the XML document.

  • <?xml → Indicates it's an XML declaration.
  • version="1.0" → Specifies the XML version being used.
  • encoding="UTF-8" → Defines the character encoding (UTF-8 is standard).

XML Comments

XML comments are notes or explanations added to the XML code that are ignored by the parser. They are used to document the code or leave messages for developers.

  • <?xml → Indicates it's an XML declaration.
  • version="1.0" → Specifies the XML version being used.
  • encoding="UTF-8" → Defines the character encoding (UTF-8 is standard).

XML Elements

XML elements can be defined as building blocks of an XML. Elements can behave as containers to hold text, elements, attributes, media objects or all of these.

XML Empty Element

An empty element in XML is an element that has no content between the start and end tags. It does not contain text or child elements.

  • <linebreak /> → It is a self-closing tag with no content inside..
  • <image /> → It is a self-closing tag with no content inside..
  • <hr /> → It is a self-closing tag with no content inside.

XML Nested Elements

Nested elements in XML are elements that are placed inside other elements to represent a hierarchical (parent-child) structure of data.

  • The XML declaration is at the top.
  • <student> → Represents the root element.
  • <course> → Represents a program element <program> and <year>.
  • All tags are properly opened and closed.
  • Indentation helps make the hierarchy clear.

XML Tags

XML tags are used to define and structure data in a readable and organized format. Unlike HTML, XML tags are not predefined, meaning users can create custom tags that describe the data they contain. These tags follow strict syntax rules, such as proper nesting and case sensitivity, to ensure consistency and compatibility across different systems and applications.

Here are some common XML Tags

Start Tag

Start tag marks the beginning of an XML element. It is written with the element name inside angle brackets.

  • A start tag marks the beginning of an XML element. It is written with the element name inside angle brackets.

End Tag

End tag marks the end of an XML element. It is written like the start tag but with a forward slash /.

  • An end tag marks the end of an XML element. It is written like the start tag but with a forward slash /.

Empty Tag

An Empty tag does not contain child elements. It combines start and end tag into a single self-closing tag.

  • <published /> and <coverImage /> are empty tags, meaning they do not contain any content between a start and end tag.

XML Attributes and Characters

XML attributes provide additional information about elements and are placed within the start tag, using a name-value pair format like attribute="value". Characters in XML refer to the text data inside elements, and special characters like <, >, and & must be written using escape codes (&lt;, &gt;, &amp;) to avoid parsing errors. Both attributes and characters must follow strict XML syntax rules to ensure well-formed and valid documents.

XML Attributes

Attributes are part of XML elements. An element can have multiple unique attributes. Attribute gives more information about XML elements. To be more precise, they define properties of elements. An XML attribute is always a name-value pair.

  • <student> → Represents the root element.
  • id, name, and course → Attributes of the student element.
  • Each attribute is written inside the start tag as name="value" pairs.
  • This element is self-closing because it doesn't contain inner content.

XML Character Entities

Character entities are special codes used in XML to represent characters that are reserved or not allowed directly in the content (like <, >, &, etc.).

  • &amp; → Represents the ampersand (&).

  • &lt; and &gt; → Represents less than (<) and greater than (>)

  • &quot; → Represents a double quote (")
